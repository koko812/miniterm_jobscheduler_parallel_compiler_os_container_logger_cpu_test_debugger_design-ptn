# ファイルシステム観察・実装プラン

目的: 本物のファイルシステム挙動を観察してから、
Python でメモリFSを自作できるようにする。

## 1. 事前観察（本物のFS）

### 1-1. 名前 → inode の対応
- 目的: ディレクトリが「名前と inode 番号の対応表」だと理解する
- コマンド例
  - `ls -li`
  - `stat ファイル名`

### 1-2. ハードリンクの挙動
- 目的: 同じ inode を複数の名前が指せることを確認
- コマンド例
  - `ln 元 もう一個`
  - `ls -li`
  - `stat 元` / `stat もう一個`

### 1-3. unlink の意味
- 目的: `rm` は「データ削除」ではなく「リンク削除」だと理解する
- コマンド例
  - `ln 元 もう一個`
  - `rm 元`
  - `cat もう一個`

### 1-4. ディレクトリ自体の inode
- 目的: ディレクトリも inode を持つことを確認
- コマンド例
  - `ls -lid .`

### 1-5. 開いている間は消えない
- 目的: 参照が残っている限り実体は消えない挙動を体験
- コマンド例
  - ターミナルA: `tail -f ファイル`
  - ターミナルB: `rm ファイル`

### 1-6. アドレス空間の観察（プロセス側）
- 目的: 仮想メモリの構造を眺める
- コマンド例
  - `cat /proc/$$/maps | head`

## 2. 自作メモリFSで模倣する要素

- ディレクトリ: 名前 → inode の対応表
- inode: `nlink / type / size / data参照`
- unlink: `nlink` を減らし、0で削除
- open中の参照: 参照カウント（簡易でOK）

## 3. Python メモリFSの最小実装

- 1階層のみ（root直下）から開始
- `create / read / write / ls / unlink / rename / link`
- 次の段階で階層ディレクトリとパス解決を追加
