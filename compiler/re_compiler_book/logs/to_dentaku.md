# ここまでのまとめ（re_compiler_book）

## パーサ・構文規則
- 左再帰は再帰下降で無限再帰になるため、`add = mul (('+'|'-') mul)*` のようにループ化して扱う。
- 典型の分解は `expr -> equality -> relational -> add -> mul -> unary -> primary`。
- `primary = num | '(' expr ')'` のとき、`(` を消費したら `expr()` を呼び、`expect(')')` で閉じる。
- `unary` は `+`/`-` を再帰で連鎖できる形（`- - - 4` など）にする。

## トークナイズ
- `token->len` を持つ設計を採用。多文字演算子や識別子・デバッグ表示で必要。
- 文字列比較は `strncmp`/`memcmp` を使う。`*p` は1文字なので `strcmp(*p, "==")` は誤り。
- `starts_with`（`is_prefix` など）を用意して `==`, `!=`, `<=`, `>=` などをまとめて処理する。
- 数字トークンの長さは `strtol` 後の `p - start` で決める。

## デバッグ
- `TRACE()` マクロでパーサ関数の入口ログを出す。
- `__func__` を使えば関数名を自動で出せる。
- 右側のカラムを揃えるなら、`namebuf` を作って `"%s:"` を幅付きで出力する方式が便利。

## スタックとフレーム
- `push rbp; mov rbp, rsp; sub rsp, N` で関数フレームを作る。
- `rbp` は関数内の固定基準点、`rsp` はスタック先端（push/popで動く）。
- `mov rsp, rbp` で一時領域（評価用 push など）をまとめて破棄できる。
- スタックに積まれるのは「データ」であり、命令自体は `.text` にある。

## 変数（LVar）
- `locals` は関数ごとのローカル変数連結リスト。
- パース時に `LVar` を作り、`offset` を割り当てておく。
- 実行時の実アドレスは `rbp - offset` で決まる（値は実行時に書き込む）。

## 浮動小数点や配列・メモリ
- VLA は実行時に `sub rsp, size` で確保される（スタック）。
- 大きい配列はヒープ（`malloc/calloc`）が安全。ヒープは遅いが柔軟。
- `vector` の中身はヒープ確保。

## 命令補足
- `set*` は `cmp` で立てたフラグを 0/1 に変換する。
- `movzb rax, al` は `al` を 0拡張して `rax` 全体を正規化する。
