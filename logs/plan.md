# 学習計画（ドラフト）

このリポジトリで「小さいターミナル / ジョブスケジューラ / コンパイラー / OS・コンテナ」を並行して遊ぶための、段階的な実験プラン。

## 1. 小さいターミナル

- 目標: プロセス・入出力・シグナルの基本を体感
- やること
- `cd`, `pwd`, `exit` などの内蔵コマンド
- `;` で複数コマンド
- `|`, `>`, `<` の最小実装
- `&`, `jobs`, `fg`, `bg` の簡易ジョブ制御再現したい
- `awk` コマンドや `cut` とか `sed` とかも再現したい（？というか使えるようになりたい）
- bash と zsh の違い？？その他近代的エディタの違い？
- 成果物: ミニシェル（短いREADME付き）

## 2. ジョブスケジューラ

- 目標: スケジューリング方針とメトリクスの理解
- やること
- ラウンドロビンのシミュレーション
- 優先度付き + aging
- EDF / RM の比較
- 失敗・再試行・バックオフの実装
- 成果物: テスト付きシミュレーター

## 3. コンパイラー

- 目標: 字句解析〜実行までの一連の流れを理解
- やること
- 変数・四則演算・if/while のミニ言語
- AST + インタプリタ
- オートマトン（状態機械）
- バイトコード生成 + VM
- 余力で最適化（定数畳み込みなど）
- 成果物: 言語仕様メモ + サンプルプログラム

## 4. 並列処理プログラミング

- 目標: 競合・同期・デッドロックを体験し、安全な並列設計を学ぶ
- やること
- スレッド/プロセスの使い分けと計測
- 共有メモリの競合再現（ロック無し → 破綻）
- ミューテックス/セマフォ/条件変数の基礎
- ワークキューでの並列化（Producer/Consumer）
- デッドロック再現と回避（順序ロック、タイムアウト）
- map/reduce, bucket の実装
- vllm (SGLang) 再現実装？？
  - (結局 vllm の最適な使い方がわかってないので)
- ray の訓練
- verl or rllib 使用 or 再現実装
- 成果物: 小さな並列タスク実行器 + 実験メモ

## 5. OS / コンテナ実験

- 目標: 隔離・制限・障害の挙動を体験
- やること
- 最小 rootfs（busybox）
- namespace 実験（PID/NET/UTS）
- cgroup で CPU/メモリ制限
- **overlayfs や破壊系テスト（disk full, OOM**
- 成果物: 実験ログと再現手順

## 6. Docker 準備: Web サーバー実験（通信 + 負荷）

- 目標: 基本的な HTTP 通信と負荷の影響を体感
- やること
- nginx / Apache コンテナを起動して疎通確認
- docker compose で `web` と `bench` を定義してまとめて起動
- ベンチマーク（`ab` / `wrk` / `hey`）で負荷テスト
- レスポンス/エラー率/遅延の観察
- 成果物: 簡単な負荷テスト結果メモ

## 7. Docker ネットワーク実験

- 目標: コンテナ間通信・分離・遅延の理解
- やること
- ブリッジネットワークで疎通・名前解決の確認（`client`, `api`, `db`）
- ルーティング確認（`ip route`, `traceroute`）
- DNS 挙動の比較（`--dns` や resolv.conf、`dnsbox` コンテナ）
- ネットワーク分離と中継（ルータ役コンテナ）
- `tc` で遅延/損失/帯域制限の注入（`netem` コンテナ）
- 成果物: 実験手順メモ + 観察ログ


## 8. 組み合わせ課題

- ミニシェルからジョブスケジューラを起動
- VM実行を cgroup 制限下で観察
- コンテナ内でミニOSをビルドして挙動を比較
