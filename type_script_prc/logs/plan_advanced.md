# スケーリング実験プラン（詳細）

目的: Node + Nginx 構成を拡張して、
スケール・耐障害・分散協調の挙動を観察する。

## 1. DB 追加（Redis / Postgres）

### 1-1. Redis（軽量・キャッシュ）
- 目的: レイテンシ・ホットデータの効果を見る
- 追加作業
  - `redis` コンテナを追加
  - Node から `get/set` するAPIを作る
- 観察ポイント
  - DBなし vs Redisありで p95/p99 の差
  - キャッシュヒット率による変化

### 1-2. Postgres（永続DB）
- 目的: DB負荷と接続プールの影響を見る
- 追加作業
  - `postgres` コンテナ追加
  - シンプルな CRUD API を作る
- 観察ポイント
  - 同時接続数とクエリ遅延
  - connection pool の効果

## 2. Node コンテナ複数化

- 目的: 横スケールの効果を観察
- 追加作業
  - Node を `replicas` で複数起動
  - Nginx でラウンドロビン
- 観察ポイント
  - RPS / レイテンシの変化
  - Nginx のボトルネック化

## 3. 負荷強度の異なる API

- 目的: 軽いAPIと重いAPIの混在時の影響を見る
- 追加作業
  - `/light` と `/heavy` を用意
  - heavyは PBKDF2 / 大きいレスポンスなど
- 観察ポイント
  - 混在時の p95/p99
  - 分離（別コンテナ）したときの効果

## 4. クラッシャー API

- 目的: 障害時の挙動を確認
- 追加作業
  - 意図的に `process.exit(1)` するAPI
  - メモリを大量に使うAPI
- 観察ポイント
  - 再起動の挙動
  - エラーレートの変化

## 5. Kubernetes 自動復旧

- 目的: 自動復旧の流れを観察
- 追加作業
  - Deployment + Service を作る
  - liveness/readiness を設定
- 観察ポイント
  - Pod再起動のタイミング
  - ヘルスチェック失敗時の挙動

## 6. 複数ノード協調

- 目的: 分散状態の共有を体験
- 追加作業
  - Node複数台 + 共有DB/Redis
- 観察ポイント
  - 整合性のズレ
  - 競合時のふるまい

## 7. Kafka 実験プラン

### 7-1. 目的
- 非同期メッセージングの挙動を体験
- バックプレッシャーと遅延を観察

### 7-2. 構成
- Kafka + Zookeeper（または KRaft）
- Producer API / Consumer API を Node で実装

### 7-3. 観察ポイント
- Producer だけ回したときの遅延
- Consumer を遅らせたときの backlog
- Consumer を増やしたときのスループット
- at-least-once の重複挙動

## 次の一歩

- まずは Redis or Node 複数化から始める
- 実験結果をログに追記する



# プロジェクト案まとめ（3テーマ）

## ① 論文管理ツール

### 目的
- 研究用に実際に使う
- 検索・整理・メモを効率化

### MVP
- 論文登録（title / authors / venue / year / url）
- タグ付け・検索
- メモ保存

### 構成
- PostgreSQL：論文・タグ・メモ（永続データ）
- Redis：検索キャッシュ
- API（node × 複数）＋ nginx

### 拡張
- PDF処理（Worker）
- embedding検索
- Kafka：`paper.created` などのイベント処理

---

## ② ゲーム基盤（リアルタイム対戦）

### 目的
- リアルタイム処理・分散状態管理の理解

### MVP
- 2人対戦（オセロ / クイズなど）
- マッチング
- ゲーム状態同期

### 構成
- Redis：ゲーム状態・マッチング・ランキング
- PostgreSQL：ユーザー・戦績
- API（node × 複数）＋ nginx

### 拡張
- WebSocketリアルタイム通信
- k8s：スケール・自動復旧
- Kafka：行動ログ・分析・リプレイ

---

## ③ 空中ホワイトボード（リアルタイム共同編集）

### 目的
- 実際に使う（リモート会話）
- CV × リアルタイム × 分散の統合

### MVP
- 手検出（MediaPipe）
- 指の軌跡で描画
- 2人でリアルタイム共有（WebSocket）

### 構成
- Redis：描画状態・presence・pub/sub
- PostgreSQL：ノート保存
- API（node × 複数）＋ nginx

### 拡張
- ジェスチャー操作
- カーソル共有
- 履歴保存
- Kafka：描画イベントログ・リプレイ

---
