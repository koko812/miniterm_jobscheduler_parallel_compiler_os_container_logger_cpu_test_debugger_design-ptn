# Redis 練習プラン（ゲーム応用を意識）

## フェーズ1: Redisの基礎
**目的**: Redisの基本操作に慣れる
- `SET / GET`（文字列）
- `DEL / EXISTS`
- `EXPIRE / TTL`（期限付き）
- `INCR / DECR`（カウンタ）

**小課題**
- 5秒で消えるキーを作る
- アクセス数カウンタを作る

## フェーズ2: ゲームっぽい状態管理
**目的**: 状態を持つ感覚を掴む
- **Hash** でプレイヤー情報
  - `HSET player:1 x 10 y 20 hp 100`
- **List** でイベントログ
  - `LPUSH match:1:log "..."`
- **Sorted Set** でランキング
  - `ZINCRBY leaderboard 1 player:1`

**小課題**
- プレイヤー位置更新API
- ランキング表示API

## フェーズ3: リアルタイム系の基本
**目的**: 即時更新の感覚を掴む
- **Pub/Sub** でイベント通知
- **TTL** で一時データ（マッチ中状態）
- **Rate Limit** で行動回数制限

**小課題**
- 攻撃イベントをPub/Subで配信
- 1秒に3回までの行動制限

## フェーズ4: 対戦ゲームに寄せる
**目的**: 試合1つ分の構造を組む
- `match:{id}` に状態を集約
- **Key設計**を考える
  - 例: `match:1:state`, `match:1:players`, `match:1:log`

**小課題**
- マッチ作成 → 参加 → 開始 → 終了 までの遷移

## 次の一歩
- まずはフェーズ1か2から着手
- 実験結果は別ログに追記
